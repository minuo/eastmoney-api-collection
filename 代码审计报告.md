## 一、安全漏洞审计

### 1.1 高危漏洞

#### A. 使用 `eval()` 函数导致的代码注入漏洞
- **文件**: `src/services/api.js:23
- **风险等级**: 🔴 高危
- **问题描述**: 使用 `eval()` 解析第三方API返回的数据，存在严重的代码注入风险
- **原始代码**:
  ```javascript
  const parsedData = eval(dataStr);
  ```
- **修复方案**: 实现了自定义解析器 `parseVarRData()`，安全解析JavaScript数组数据
- **修复状态**: ✅ 已修复
- **修复代码**:
  ```javascript
  const parseVarRData = (text) => {
    const dataStr = text.replace(/^var r =/, '').replace(/;$/, '');
    // 自定义安全解析器
    // ...
  };
  ```

#### B. XSS（跨站脚本攻击）漏洞
- **文件**: `src/views/Notes.vue:62
- **风险等级**: 🔴 高危
- **问题描述**: 使用 `v-html` 直接渲染 Markdown 转译后的 HTML，存在 XSS 风险
- **原始代码**:
  ```javascript
  const renderedContent = computed(() => {
    if (!currentNote.value?.content) return ''
    return marked(currentNote.value.content)
  })
  ```
- **修复方案**: 引入 `DOMPurify` 库，安全清理 HTML
- **修复状态**: ✅ 已修复
- **修复代码**:
  ```javascript
  import DOMPurify from 'dompurify'
  const renderedContent = computed(() => {
    if (!currentNote.value?.content) return ''
    const html = marked(currentNote.value.content)
    return DOMPurify.sanitize(html)
  })
  ```
